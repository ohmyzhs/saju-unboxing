# 사주 분석 프롬프트 (경쟁사 역설계 → 우리 우위 설계)

이 문서는 `api/_lib/analysis.js`의 `saju-analysis` 기본 프롬프트와 `ANALYSIS_SCHEMA`,
그리고 `api/_lib/sajuContext.js`(만세력 컨텍스트 빌더)의 설계 근거를 정리한다.

---

## 1. 경쟁사 프롬프트 역설계 (리포트 2건 교차분석)

분석 표본: 동일 포맷으로 생성된 두 사람의 리포트.
- 김진호 1989-08-22 양력 10:00 여 (甲寅일주)
- 김미소 1989-09-06 양력 10:00 여 (己巳일주)

추정되는 경쟁사 프롬프트의 특징:

1. **고정 13섹션 템플릿.** 두 리포트의 섹션 순서·역할·정서 비트가 거의 동일하다.
   (총평→오행→일주/십성→단점경고→위로공감→성격입체→직업→재물→연애→가족→인맥→이동→마무리개운)
2. **제목 = 자연·날씨·풍경 은유** ("천둥번개 치는 벌판의 거목", "불타는 대지 위 오아시스").
3. **본문에 명리 전문용어를 그대로 노출**(간여지동·삼형살·갑기합→토·식상생재·생지·투출)해 권위를 만든다.
4. **위로/치유 섹션을 통째로 1개** 둔다("아무도 알아주지 않는 외로운 싸움…").
5. **반복 상투구**가 두 리포트에 거의 같은 문장으로 등장(바넘효과).
6. **결정적 약점 — 개운책 하드코딩.** 두 사람에게 행운수(1·6)·색(검정/푸른)·방향(북쪽)을
   **동일하게** 줬다. 이는 "더운 사주(화·토 많음)는 무조건 水로 식혀라"는 조후 단일 규칙을
   박아둔 것으로, 신강약·억부·진짜 용신을 계산하지 않았다는 강한 정황.

**강점(흡수할 것):** 은유 제목 / 전문용어 권위 / 위로 정서 / 풍부한 섹션 수.
**약점(이길 지점):** 사람마다 안 달라지는 개운책·상투구(바넘), 데이터 미활용.

## 2. 만세력 대조 — 우리 API가 더 정확하고 더 풍부

우리 중앙 만세력 API(`/api/v1/saju/compute`)로 동일 두 생일을 계산한 결과,
경쟁사 표의 모든 사실(사주 4기둥·오행분포·십성·12운성·형충회합)이 **우리 값과 100% 일치**.
즉 사실은 동일하고, **차이는 해석(프롬프트) 레이어에서만 발생**한다.

게다가 우리 API는 경쟁사 표에 없는 데이터를 추가로 준다:
- **용신 4단계** (용신/희신/기신/구신/한신) — 경쟁사는 단일 조후만.
- **형충회합 전체** (천간합/충/병존, 지지 삼합/방합/육합/충/형/파/해/암합).
- **십성 통계** (비겁/식상/재성/관성/인성 개수) → 과다·공백 진단.
- **지장간·납음·대운 10주기·올해 일진**.

### 교차검증 결론 (스모킹건)

| 항목 | 김진호 | 김미소 |
|---|---|---|
| 신강약 | 중신약 | 약변강 |
| 오행 | 균형(과다·공백 없음) | **화3·토3 과다 + 목 0(공백)** |
| 십성 | 균형 | **인성3 과다 + 관성 0(공백)** |
| 형충회합 | 寅巳申삼형·寅申충 등 9건(역동) | 巳申형·파 등 5건(삼형 없음) |
| 용신/희신 | 木 / 水 | 木 / 水 |
| 대운(현재) | 35세 식신 | 31세 정인 |

두 사람은 **용신이 같다(木)**. 그래서 색/숫자/방향만 용신에서 뽑으면 둘 다 초록·동쪽으로
**또 똑같아진다.** 진짜 차별화는 용신 한 글자가 아니라 **구조 전체**에서 나온다:
오행공백(미소=목 공백)·십성공백(미소=관성 0)·형충 유무·신강약·대운 십성.
→ 프롬프트가 이 구조 차이를 강제로 서사에 반영해야 바넘을 탈출한다.

## 3. 우리 적용 설계

### (a) LLM 입력 강화 — `buildSajuContext(manse)`
기존엔 `summary`만 넘겨 형충회합·용신4단계·십성통계·지장간을 못 썼다.
이제 `summary + full`을 병합한 통합 컨텍스트를 넘긴다. 핵심 필드:
`대상(일간·신강약)`, `원국(4기둥: 십성·12운성·신살·지장간·납음)`, `오행분포/과다/공백`,
`십성통계/과다/공백`, `용신체계(4단계)`, `개운파생근거(용신·희신→색/숫자/방향)`,
`형충회합(한국어 라인)`, `대운(현재+10주기)`, `올해`, `공망`, `천을귀인`.

### (b) 개운 파생 규칙 (코드가 확정 — 모델 환각·바넘 차단)
색/숫자/방향/본명수/보조기운은 **LLM이 아니라 `deriveLucky()`(sajuContext.js)가 결정적으로 계산**한다.
모델은 산문(`why`·`whyKeywords`·`personalNote`)만 쓰고, 서버가 응답을 받은 뒤 사실값을 덮어쓴다.
- 색/방향: `ELEMENT_REMEDY`(목 초록·청록/동, 화 빨강/남, 토 노랑/중앙, 금 흰색/서, 수 검정·청/북)에서 용신·희신으로.
- **본명수(numberFocus)**: 용신 오행의 河圖 수쌍(목=3·8) 중 **일간 음양**으로 1개 강조 → 같은 용신이라도 갈린다.
  (예: 김진호 일간 甲(양)→ 3 강조 / 김미소 일간 己(음)→ 8 강조)
- **무근 플래그(rootless)**: 용신 오행을 한 글자도 못 타고났으면 true → 방향 표기를 "특히 의식"으로 바꾸고,
  프롬프트가 "타고난 성향으로 단정하지 말고 외부 보완으로만 권함" 분기.
- 이로써 용신이 같아도 두 사람의 개운 카드(본명수·방향표기·무근·why·personalNote)가 동일해지지 않는다.

### (c) 사주별 분기 지시 (프롬프트 내장)
신강/신약, 오행공백 유무, 십성공백(특히 관성0), 형충(삼형·충) 유무, 대운 십성에 따라
서사 톤과 핵심 테마가 갈리도록 조건 분기를 명시.

### (d) 섹션 플랜 — 경쟁사 13개를 베끼지 않되 7~10개
경쟁사 주제를 더 적은 수로 통합하고, **우리 데이터로만 가능한 섹션**을 추가:
타고난 기질/그릇, 오행 지형(과다·공백), 십성의 빛과 그림자, **형충이 만드는 인생 변동선**,
일과 돈, 관계의 온도, **대운 타임라인과 올해**, 마지막 개운법(+위로 1자리).
경쟁사 강점(은유 제목·전문용어 권위·위로)은 흡수, 상투구·바넘은 배제.

---

### (c) 사주별 분기 강화 (finalize 적대검증 반영)
경계 신강약(약변강·중신약)·간여지동·병존·관성0 vs 미약·무근용신·희신 조후충돌 분기를 프롬프트에 명문화.
신살(건록·역마살·문창귀인·양인살·천을귀인 등)과 지장간을 인용해 권위를 살리되, 신살은 길흉 단정이 아닌
기질·재능 단서로만 쓰고, 표면상 0인 오행/십성도 지장간에 숨었으면 "겉으론 비었지만 속에 품은 기운"으로
해석해 결핍을 과장하지 않는다(겁주기 방지). 경쟁사 상투구 6종은 네거티브 목록으로 금지.

## 4. Output Schema (`ANALYSIS_SCHEMA`, strict)

```json
{
  "headline": "이 사주만의 한 줄(본인 글자 인용)",
  "sections": [
    { "icon": "이모지", "title": "사주 글자가 은유로 녹은 제목",
      "titleAnchor": "제목이 인용한 글자/개념 1개(예: 寅申충, 관성0, 간여지동)",
      "body": "3~4문단: 해석→근거(글자·용어)→구조·비유→실천조언",
      "evidence": ["寅申충", "중신약", "관성0"] }
  ],
  "lucky": {                         // 모델은 아래 산문만 출력
    "why": "왜 이 용신이 필요한지(오행공백·십성과다·신강약 중 2개 인용, 희신 조후 과잉 주의)",
    "whyKeywords": ["목공백", "관성0"],
    "personalNote": "이 사람만의 한 줄(형충·간여지동·병존·공백 인용)"
  }
}
```
- 섹션 7~10개, `titleAnchor`+`evidence`로 만세력 근거 강제(프론트에서 근거 칩 표시).
- `lucky`의 사실값(element/color/number/numberFocus/direction/assist/rootless)은 **서버가 `deriveLucky`로 채워 병합** → LLM 환각 불가.
- 다른 상품(궁합/대운/연도운/오늘운세)은 기존 `REPORT_SCHEMA`/`DAILY_SCHEMA` 유지.
- 응답 후 **공포·단정·투자 어휘 백스톱**(`hasFearWords`): 적발 시 1회 재작성.

## 5. Style Rules
- **말투·표기(최우선):** 쉬운 일상어 우선. 한자는 단독 노출 금지 — 한글 음을 먼저 쓰고 괄호 보조만(갑인일주(甲寅), 인신충(寅申)). 전문용어는 섹션당 2~3개로 제한+즉시 풀이. 지장간·납음·위치강약은 재료로만 쓰고 나열 금지. 감정·공감 먼저. (경쟁사가 한글 우선인데 우리가 한자를 쏟아내 더 어려워 보였던 문제를 교정.)
- 결론 먼저 → 만세력 근거(실제 글자·용어). "사주에 따르면"식 두루뭉술 금지.
- 제목은 매번 새로 + `titleAnchor`에 인용 글자 명시(일반 자연 풍경만으로 짓기 금지).
- "반드시/무조건/큰일 난다/암/이혼/파산/대박" 공포·단정·투자 어휘 금지. 리스크는 관리법으로 전환.
- 위로도 사주 근거에 연결(막연한 위로 금지). 경쟁사 상투구 6종 금지.
- 사람마다 결과가 달라야 한다(바넘 탈출) — 교차검증: 김진호↔김미소 섹션 제목 겹침 0/8, 본명수 3↔8 확인됨.
