# 자체 통합 만세력 엔진 설계

## 목표

SAJULAB 만세력 API 호출을 제거하고 현재 애플리케이션 안에서 만세력 데이터를
계산한다. 기존 리포트가 사용하는 `summary`와 `full` 계약은 유지하면서 원국
정확도는 한국식 음력과 KASI 기준으로 높이고, 출력 정보는 기존 SAJULAB과
동등하거나 더 자세하게 제공한다.

계산은 고정 한국표준시(UTC+09:00)를 사용한다. 출생지, 경도, 균시차,
진태양시, 역사적 서머타임 보정은 적용하지 않는다.

## 두 엔진 비교와 결정

`yhj1024/manseryeok`은 한국식 음양력 변환, 절기, 사주 원국, 십신, 공망,
대운을 Node.js에서 직접 계산한다. KASI 골든 데이터와 한국·중국 음력 날짜가
갈리는 사례를 검증하고 있어 원국 계산의 기준으로 적합하다.

로컬 `saju_engine`은 `lunar-python`을 원국 기반으로 사용하고 KASI/eacal
절기를 교차검증한다. 지장간, 신강약, 용신 체계, 형충회합, 납음, 십이운성,
신살, 대운·세운·월운 등 리포트용 부가 계산은 훨씬 풍부하다. 다만 한국과
중국의 음력 날짜가 갈리는 일부 입력에서는 원국이 하루 어긋나고, 현재 기본
평균태양시 보정은 고정 한국표준시 정책과 맞지 않는다.

따라서 다음 혼합 방식을 선택한다.

- 통합 엔진의 출력 구조와 부가 해석 규칙은 로컬 엔진을 기준으로 한다.
- 음양력 변환, 절기 경계, 연·월·일·시주는 `manseryeok` 결과만 사용한다.
- 로컬 계산기는 확정된 원국을 입력받아 부가 데이터만 계산한다.
- 서로 다른 두 원국을 계산한 뒤 임의로 선택하거나 섞지 않는다.
- 현재 Node/Vercel 애플리케이션에 맞춰 로컬 Python 계산기를 JavaScript로
  이식한다. 별도 Python 서비스나 내부 HTTP 호출은 만들지 않는다.

## 대안과 선택 이유

### `manseryeok` 단독

배포와 원국 정확도는 가장 단순하지만 신강약, 용신 5단계, 신살,
형충회합, 세운·월운 등 현재 리포트에 필요한 정보가 부족하다.

### 로컬 Python 엔진 단독

출력은 풍부하지만 한국식 음력 예외를 별도로 고쳐야 한다. Node와 Python
런타임을 함께 운영하면 서버리스 콜드 스타트, 패키징, 내부 호출 실패 지점이
늘어난다.

### Node 통합 엔진

`manseryeok`을 달력 커널로 사용하고 로컬 계산기를 JavaScript로 이식한다.
정확도, 데이터 범위, 현재 배포 구조, 응답 속도를 함께 만족하므로 이 방식을
채택한다.

## 전체 구조

```text
프로필 입력
  -> 입력 정규화 및 검증
  -> manseryeok 달력 커널
  -> 표준 원국 모델
  -> 로컬 규칙 기반 상세 계산기
  -> SAJULAB 호환 어댑터
  -> 기존 리포트 컨텍스트와 저장 로직
```

### 입력 정규화

프로필을 다음 규칙으로 표준 입력으로 바꾼다.

- `solar`, `lunar`, `leap_lunar`를 명시적으로 구분한다.
- 성별 `M/F`를 `male/female`로 변환한다.
- 입력 시각은 항상 고정 UTC+09:00의 벽시계 시각으로 해석한다.
- 진태양시와 경도 보정 옵션은 전달하지 않는다.
- `yajasi=no`는 자정 기준인 `midnight`를 사용한다.
- `yajasi=yes`는 23시부터 다음날로 보는 `jasi`를 사용한다.
- 정확한 시각 입력은 분 단위 값을 사용한다. 시간대 선택 입력은 현재 UI가
  저장하는 해당 시진의 중앙 시각을 사용하고 `timePrecision=period`를 남긴다.
- 음력 윤달이 실제 달력에 존재하지 않으면 입력 오류로 처리한다.

`manseryeok` 버전은 정확히 고정한다. 의존성 업데이트는 골든 테스트와 전체
회귀 테스트를 통과한 경우에만 허용한다.

### 달력 커널과 표준 원국

달력 커널은 다음 사실값을 확정한다.

- 양력·음력 변환 결과와 윤달 여부
- 적용된 일 경계 정책
- 연주, 월주, 일주, 시주
- 각 천간·지지의 오행과 음양
- 십신, 공망, 대운 방향과 시작 시점
- 가장 가까운 절기와 절입 경계 경고

표준 원국은 한글과 한자를 모두 보존한다. 이후 계산기는 달력이나 원국을
다시 계산하지 않고 이 표준 모델만 입력으로 받는다.

### 상세 계산기

로컬 엔진의 규칙과 상수를 JavaScript로 이식해 다음 데이터를 계산한다.

- 주별 지장간과 천간·지지 십성
- 오행 분포와 음양 분포
- 십성 5그룹 통계
- 실령·실지·득세 기반 신강약 5단계와 신뢰도
- 부억법·조후법 기반 용신, 희신, 기신, 구신, 한신과 후보 목록
- 천간합·충, 지지 육합·삼합·방합·반합·암합·충·형·파·해
- 네 기둥의 납음과 십이운성
- 주별 신살, 전체 신살, 근거 위치, 길흉 분류와 의미
- 대운 10주기, 현재 대운, 향후 세운, 올해 월운
- 각 운의 간지, 십성, 십이운성, 지장간, 납음과 주요 신살

신강약, 용신, 일부 신살처럼 학파에 따라 결과가 달라지는 값은 사실값과
구분한다. 각 결과에 `method`, `rulesetVersion`, `confidence`, `reasons`를
포함하고, 낮은 신뢰도나 특수격 가능성은 숨기지 않는다.

## 시간 미상 처리

시간을 모르는 입력에 정오를 실제 출생 시각처럼 사용하지 않는다.

- 시주는 `null`로 반환한다.
- 시주 지장간, 시주 십성, 시주 신살 등 직접 의존 항목은 산출 보류한다.
- 오행 분포는 6글자, 십성 통계는 확인 가능한 5개 대상만 집계하고
  `countBasis`를 함께 반환한다.
- 같은 날짜의 00:00과 23:59를 계산해 연주·월주·일주가 동일한지 확인한다.
- 절입일이라 월주나 연주가 달라질 수 있으면 하나를 임의로 선택하지 않고
  후보 원국과 `boundaryAmbiguous=true`를 반환한다.
- 대운 시작 나이는 단일값 대신 산출 가능한 범위를 제공한다. 현재 대운이
  범위 전체에서 같을 때만 확정하고, 다르면 후보로 반환한다.
- 시주 의존도가 높은 해석에는 `uncertainFields`를 전달해 LLM이 단정하지
  않도록 한다.

## 응답 계약

`computeManse(profile)`의 외부 계약을 유지한다.

```json
{
  "ok": true,
  "cost": 0,
  "summary": {},
  "full": {}
}
```

`summary`는 기존 소비 코드가 직접 사용하는 다음 항목을 보장한다.

- `dayStem`, `dayBranch`, `yearAnimal`, `strength`, `yongsin`
- `elements`, `pillarsDetail`
- `daeunList`, `currentDaeun`, `thisYear`
- `shinsal`

`full`은 다음 기존 항목을 보장한다.

- `pillar.year/month/day/hour`
- `tenGodStats`, `strength`, `yongsin`, `hyungchung`, `shinsal`
- `daeun`, `nyunun`, `wolun`

로컬 엔진의 배열형 지장간은 기존 계약에 맞춰
`yeogi/junggi/bongi`로 변환한다. `관살`은 기존 리포트 계약의 `관성`으로
매핑하고, 오행 영문 키는 한국어 키로 변환한다.

기존 필드 외에 다음 메타데이터를 추가한다.

- 엔진 이름과 버전
- 달력 커널과 상세 규칙 버전
- 적용 시각 정책과 일 경계 정책
- 시간 정확도와 절기 경계 경고
- 학파 의존 계산의 방법·신뢰도·근거
- 산출 보류 및 불확실 필드

기존 `buildSajuContext`, `buildCompatContext`, `buildCycleContext`,
`buildYearlyContext`는 새 응답을 그대로 소비할 수 있어야 한다. 리포트 저장
구조와 프론트 응답 구조도 변경하지 않는다.

## 데이터 우선순위

충돌을 방지하기 위해 데이터 출처 우선순위를 고정한다.

1. 음양력, 절기, 원국, 일 경계는 `manseryeok`을 사용한다.
2. 원국에서 결정적으로 파생되는 지장간, 십성, 납음, 십이운성 등은 이식한
   로컬 규칙을 사용한다.
3. 신강약, 용신, 신살 등 학파 의존 결과는 로컬 규칙과 버전을 명시한다.

상위 단계의 사실값과 하위 계산이 모순되면 결과를 조용히 반환하지 않고
계산 오류로 처리한다.

## 오류 처리

- 날짜 형식, 성별, 음력 윤달, 지원 범위를 요청 검증 단계에서 확인한다.
- 사용자 입력 오류는 기존 API 처리 방식에 맞는 한국어 메시지와 400 계열
  상태로 반환한다.
- 달력 커널 실패나 내부 계약 불일치는 500 계열 계산 오류로 반환한다.
- 상세 필드가 누락됐다고 임의값을 만들거나 SAJULAB으로 자동 대체하지 않는다.
- 시간 미상과 절기 경계는 오류가 아니라 명시적인 불확실 결과로 반환한다.
- 운영 응답에 내부 스택이나 민감한 설정을 노출하지 않는다.

## 전환 범위

- `computeManse` 내부를 외부 API 호출에서 로컬 통합 계산으로 교체한다.
- 만세력 API 기본 주소, 키, 상품 코드와 포인트 잔액 의존성을 제거한다.
- 관리자 만세력 API 설정과 연결 테스트를 제거하거나 자체 엔진 상태 표시로
  바꾼다.
- 헬스 체크와 설치 문서에서 SAJULAB 환경변수를 제거한다.
- 기존 분석, 궁합, 대운, 연도운, 무료 일일운세의 호출부는 유지한다.
- 기존 저장 데이터는 읽을 수 있게 두고 새 계산 결과부터 엔진 메타데이터를
  저장한다.

## 테스트와 완료 조건

제품 코드보다 테스트를 먼저 작성한다.

### 원국 정확도

- `manseryeok`의 KASI 골든 원국 11건을 고정 회귀 테스트로 둔다.
- 음력 1997-01-01, 2023-04-01, 2049-12-01 등 한국식 음력 예외를 검증한다.
- 입춘과 월 절입 전후의 연주·월주를 검증한다.
- 23:30 입력에서 야자시 적용 여부에 따른 일주·시주 차이를 검증한다.
- 1992-10-24 05:30 입력이 평균태양시로 이동하지 않고 한국표준시 시주를
  유지하는지 검증한다.
- 양력·음력·윤달 왕복 변환 표본을 검증한다.

### 상세 데이터

- 지장간, 십성, 납음, 십이운성의 알려진 표본을 검증한다.
- 신강약과 용신은 로컬 엔진의 현재 규칙 표본과 동일해야 한다.
- 합충형파해, 삼합·방합·반합·암합의 규칙 표본을 검증한다.
- 신살은 이름뿐 아니라 근거 위치, 분류, 의미를 함께 검증한다.
- 대운 10개, 향후 세운, 12개월 월운과 각 상세 필드를 검증한다.

### 호환성과 불확실성

- 기존 네 컨텍스트 빌더가 모든 상품에 필요한 비어 있지 않은 데이터를
  생성하는지 검증한다.
- `summary/full` 필드 계약을 스냅샷과 명시적 단언으로 검증한다.
- 시간 미상 시 시주가 없고 6글자/5대상 집계 기준이 표시되는지 검증한다.
- 절입일 시간 미상 시 후보 원국과 경계 경고가 반환되는지 검증한다.
- 계산 중 외부 만세력 HTTP 호출이 발생하지 않는지 검증한다.

완료 조건은 다음과 같다.

- 기존 리포트 네 종류와 무료 일일운세가 새 엔진 응답으로 생성된다.
- 기존 컨텍스트가 사용하던 SAJULAB 필드가 누락되지 않는다.
- 로컬 엔진이 제공하던 상세 계산 결과와 근거 메타데이터가 포함된다.
- 외부 만세력 API 키 없이 동작하며 응답 `cost`는 0이다.
- 일반 입력의 자체 계산은 외부 네트워크 왕복 없이 수행된다.
- JavaScript 문법 검사와 전체 자동화 테스트가 통과한다.

## 범위 밖

- 출생지 입력과 국가별 시간대 지원
- 진태양시, 경도, 균시차, 역사적 서머타임 보정
- 새로운 명리 학파 또는 격국 판정 추가
- 기존 리포트 문체와 섹션 분량 변경
- 과거에 저장된 SAJULAB 결과의 일괄 재계산
