<a name="0"></a>
## 0. 이 강의로 뭘 만드나요?

**"사주연구소"** 라는 모바일 사주 웹사이트를 만듭니다. 이건 **진짜로 작동하는** 사이트예요.

- 홈 / 사주 등록 / 궁합 / 대운 / 택일 / 오늘 운세 / 대화방 / 보관함
- **진짜 사주 분석** — 생년월일을 넣으면 실제 만세력(사주 원국)을 계산하고, **AI가 해설 글**을 써줍니다.
- **카카오 로그인** (고객이 카카오로 로그인)
- **토스 결제** (실제 카드 결제창)
- **관리자 페이지** (매출·방문·결제 통계 + 상품·가격·프롬프트·이미지를 직접 수정)

### 이 사이트의 큰 그림 (중요!)

이 사업은 **"본사(대표) + 가맹점(나)"** 구조예요. 프랜차이즈 카페를 생각하면 쉬워요.

- **본사(대표)** = 원두를 공급하는 곳. 여기서는 **"만세력 계산"** 이라는 핵심 재료를 **중앙 API**로 공급합니다.
- **가맹점(나)** = 내 가게. 내가 받은 **API 키**로 본사에 만세력을 요청하고, 요청할 때마다 **포인트가 차감**돼요.
- 나머지(내 사이트 호스팅, 데이터 저장, 결제, AI 해설, 로그인)는 **전부 내 것**으로 독립 운영합니다.

| 구성 | 사용 기술 | 쉽게 말하면 | 누구 것 |
|------|-----------|-------------|---------|
| 화면 (보이는 부분) | HTML, CSS, JavaScript | 가게 **인테리어·메뉴판** | 나 |
| 서버 (뒤에서 일하는 부분) | Vercel 서버리스 함수 | 주문 처리하는 **계산대** | 나 (Vercel) |
| 데이터 저장 | Supabase | 장부를 적는 **금고·창고** | 나 (Supabase) |
| **만세력 계산** | 중앙 API (대표) | 핵심 재료 **원두 공급** | **본사(대표)** |
| AI 해설 글 | OpenAI | 글 써주는 **카피라이터** | 나 (OpenAI 키) |
| 로그인 | 카카오 | **신분증 확인 창구** | 나 (카카오 앱) |
| 결제 | 토스페이먼츠 | **카드 단말기** | 나 (토스 키) |

> 💡 이 프로젝트는 **React 같은 복잡한 도구를 안 씁니다.**
> 가장 기본인 HTML/CSS/JS + 작은 서버 함수만 써서, 입문자가 "코드가 어떻게 도는지" 이해하기 좋아요.

> 💰 **돈 흐름 한 줄 요약**: 고객 결제(토스) → 내 수입. 만세력 호출 → 내 포인트 차감(본사에 지불). AI 해설 → 내 OpenAI 비용.
> 그래서 **내 마진 = 고객결제 − 포인트 − AI비용** 입니다.

## 1. 코딩 기초 — 웹사이트는 어떻게 만들어질까?

### 1-1. 웹사이트는 3가지 재료로 만들어져요

요리에 비유하면:

- **HTML** = 음식의 **재료와 뼈대** (제목, 글, 버튼, 이미지를 "배치")
- **CSS** = 음식의 **플레이팅·데코** (색, 크기, 위치)
- **JavaScript(JS)** = 음식의 **움직임·맛** (버튼 누르면 뭔가 일어나게)

### 1-2. 프론트엔드 vs 백엔드

식당으로 비유:

- **프론트엔드** = **홀(손님 공간)**. 우리의 `index.html`, `styles.css`, `app.js`
- **백엔드** = **주방(안 보이는 공간)**. 우리의 `api/` 폴더 안 작은 함수들

손님이 "분석해줘"(프론트) → 주방에서 본사에 만세력 받아오고 + AI로 글 쓰고 + 금고에 기록(백엔드).

### 1-3. 서버 / 서버리스란?

**서버 = 손님 요청을 처리하는 컴퓨터.** 옛날엔 "24시간 켜둔 컴퓨터 한 대"였어요.

우리는 **Vercel의 "서버리스(serverless)"** 를 씁니다.
→ 평소엔 꺼져 있다가, **요청이 올 때만 잠깐 켜져서 일하고 다시 꺼지는** 방식이에요.
직접 컴퓨터를 켜둘 필요가 없어서 **싸고 관리가 쉽습니다.** (`api/` 폴더의 파일 하나하나가 작은 일꾼)

### 1-4. 데이터베이스(DB)란?

**데이터베이스 = 정보를 차곡차곡 적어두는 디지털 장부.** 회원·결제·방문·분석결과를 저장해요.

우리는 **Supabase**(클라우드 데이터베이스)를 씁니다.
서버리스는 "켜졌다 꺼졌다" 하니까, 기억(데이터)은 **항상 켜진 창고(Supabase)** 에 맡겨야 사라지지 않아요.

## 2. 용어 사전 — 어려운 말 쉽게 풀기

| 용어 | 쉬운 비유 |
|------|-----------|
| **Git / GitHub** | 코드의 **저장 기록 + 인터넷 창고**(코드계의 구글 드라이브) |
| **터미널 / PowerShell** | 마우스 대신 **글자(명령어)로 시키는 채팅창** |
| **Node.js** | 자바스크립트를 **컴퓨터에서 돌게 해주는 엔진** |
| **npm** | 남이 만든 코드 부품을 **설치하는 앱스토어** |
| **Vercel** | 내 사이트를 **인터넷에 올려 24시간 돌려주는 서비스**(호스팅집) |
| **서버리스 / 함수** | 요청 올 때만 **잠깐 켜졌다 꺼지는 작은 일꾼** (`api/` 안 파일들) |
| **Supabase** | 회원·결제·방문기록을 저장하는 **항상 켜진 창고(데이터베이스)** |
| **OpenAI** | **AI에게 글을 쓰게 시키는 곳**. 여기 키로 사주 해설을 생성 |
| **API** | 가게의 **주문 창구**. "이렇게 요청하면 이렇게 응답" 약속 |
| **API 키** | 그 창구를 쓸 수 있는 **출입증/비밀번호** (대표가 발급) |
| **포인트** | 만세력 호출 **이용권**. 한 번 부를 때마다 차감(대표 pricing에서 관리) |
| **환경변수 (.env)** | 비밀번호·열쇠를 적어두는 **금고 쪽지** (코드에 직접 안 쓰고 숨김) |
| **localhost (로컬호스트)** | **내 컴퓨터 자신** 주소. `localhost:3000` = "내 컴퓨터의 3000번 문" |
| **배포(deploy)** | 만든 걸 **인터넷에 정식 오픈**하는 것 (개업!) |
| **도메인** | 사이트의 **간판 주소** (예: `saju-lab.com`) |
| **OAuth (카카오 로그인)** | 카카오가 "이 사람 맞아요" **대신 인증해주기** |
| **Claude Code** | 터미널에서 **AI에게 코딩을 시키는 도구** (이 강의의 핵심 무기) |

## 5. 프로젝트 파일 구조 이해하기

```
saju_make/
├── index.html        ← 사용자 화면 뼈대 (홈, 사주등록, 궁합 등)
├── styles.css        ← 디자인(꾸미기)
├── app.js            ← 사용자 화면 움직임 (버튼/결제/분석 호출)
│
├── admin.html        ← 관리자 페이지 뼈대
├── admin.js          ← 관리자 움직임 (상품·프롬프트·이미지·통계)
├── setup.html        ← 설정 점검 페이지 (/setup, ✅/❌ 확인)
│
├── api/              ← 백엔드(주방). 서버리스 함수들
│   ├── saju/analyze.js     ← 중앙 만세력 호출 + AI 해설 생성 (핵심!)
│   ├── orders.js, payments/confirm.js  ← 토스 결제
│   ├── auth/kakao/...       ← 카카오 로그인
│   ├── admin/...            ← 관리자 로그인·설정저장·이미지업로드·통계
│   ├── track.js, config.js, session.js, health.js
│   └── _lib/                ← 공용 부품 (supabase, openai, toss, kakao 등)
│
├── supabase/schema.sql   ← Supabase에 1회 붙여넣을 "창고 설계도"
├── vercel.json           ← Vercel 설정
├── package.json          ← 프로젝트 설명서 + 부품 목록
├── .env                  ← 🔑 비밀키 적는 금고 (절대 공개 금지!)
└── .gitignore            ← GitHub에 안 올릴 목록(.env 포함)
```

> ⚠️ **`.env`(비밀키)는 절대 GitHub에 올리면 안 됩니다.** `.gitignore`에 적혀 있어 자동으로 안 올라가요.
> (집 비밀번호를 인터넷에 공개하면 안 되는 것과 같음)

## 6. 내 컴퓨터에서 사이트 실행하기 (로컬)

### 6-1. 폴더 열기
받은 `saju_make` 폴더를 바탕화면에 두고 → VS Code → `파일 > 폴더 열기` → `saju_make` 선택.

### 6-2. 터미널 열기
VS Code 메뉴 `터미널 > 새 터미널`.

### 6-3. 부품 설치 + Vercel 설치 (처음 1회)
```powershell
npm install
npm i -g vercel
```

### 6-4. 서버 켜기
```powershell
vercel dev
```
처음 실행하면 Vercel 로그인/프로젝트 연결을 물어봐요. 안내대로 Enter 위주로 진행하면 됩니다.

### 6-5. 브라우저에서 확인
- 사용자 화면: **http://localhost:3000**
- 설정 점검: **http://localhost:3000/setup** ← 여기서 ✅/❌ 확인 (가장 중요!)
- 관리자: **http://localhost:3000/admin**

> 💡 화면은 떠도 **분석은 7장(.env)** 을 채워야 진짜로 작동합니다.
> `/setup`이 빨간 ❌를 보여주면 그 항목을 채우라는 뜻이에요.

### 6-6. 끄기 / 다시 켜기
- 끄기: 터미널에서 `Ctrl + C`
- 다시: `vercel dev`
- **`.env`를 고치면 반드시 껐다 켜야** 반영됩니다.

## 7. 열쇠(키) 6개 연결하기 — .env 채우기

분석·결제·로그인을 진짜로 작동시키려면 **비밀키**가 필요해요. 폴더의 **`.env`** 파일을 열고 **위쪽 6칸만** 채웁니다.

```
SAJU_API_BASE=        ← (1) 대표에게 받은 중앙 서버 주소
SAJU_API_KEY=         ← (2) 대표에게 받은 API 키
SUPABASE_URL=         ← (3) Supabase 주소
SUPABASE_SERVICE_KEY= ← (4) Supabase 서비스 키
OPENAI_API_KEY=       ← (5) OpenAI 키
KAKAO_REST_API_KEY=   ← (6) 카카오 REST API 키 (로그인 쓸 때만)
```

### 7-1. (1)(2) 중앙 만세력 API — 대표에게 받기
대표가 운영하는 **만세력 서버 주소**와 **내 가맹점 API 키**를 받아 붙여넣습니다.
호출할 때마다 포인트가 차감되고, 포인트는 대표가 `sajulab.kr/admin/pricing`에서 관리해요.

### 7-2. (3)(4) Supabase — 데이터 창고 만들기
1. https://supabase.com 가입 → **New Project** (DB 비밀번호 정하고 메모)
2. 왼쪽 **SQL Editor** → **New query** → 폴더의 `supabase/schema.sql` **전체 복사 → 붙여넣기 → RUN**
   (창고에 선반을 짜 넣는 단계예요)
3. **Project Settings → API**:
   - `Project URL` → **SUPABASE_URL**
   - `service_role` (secret) → **SUPABASE_SERVICE_KEY**

### 7-3. (5) OpenAI — AI 해설 키
https://platform.openai.com → 로그인 → **API keys** → **Create new secret key** → 복사해서 **OPENAI_API_KEY**.
(모델은 기본 `gpt-5.4-mini`. 더 좋은 품질을 원하면 `.env`의 `OPENAI_MODEL`을 `gpt-5.5`로.)

### 7-4. (6) 카카오 로그인 (선택) — ⚠️ 함정 많음, 순서대로!

> 안 해도 사이트는 다 작동합니다(로그인 버튼만 꺼짐). 토스 심사에서 로그인을 요구하면 켜세요.
> 카카오는 **"4겹"이 다 맞아야** 됩니다. 하나라도 빠지면 에러가 나요.

1. https://developers.kakao.com → **애플리케이션 추가** (앱 이름·사업자명)
2. **앱 설정 > 앱 > 플랫폼 키** → **REST API 키** 복사 → `KAKAO_REST_API_KEY`
3. **① 활성화**: 제품 설정 > **카카오 로그인 > 일반** → **활성화 ON** (안 하면 **KOE004**)
4. **② Redirect URI 등록**: **앱 > 플랫폼 키 > REST API 키** 페이지의 "카카오 로그인 리다이렉트 URI"에 등록 (안 맞으면 **KOE006**)
   ```
   https://본인주소.vercel.app/api/auth/kakao/callback
   https://본인도메인/api/auth/kakao/callback        ← 커스텀 도메인 쓰면 이것도
   ```
   ⚠️ 경로가 **`/api/auth/kakao/callback`** (앞에 `api` 들어감!), `www.` 유무까지 정확히.
5. **③ Web 플랫폼 도메인**: 앱 > 플랫폼 키 > **Web** → 사이트 도메인 등록 (`https://본인주소...`)
6. **④ 클라이언트 시크릿 끄기** ⭐ (안 끄면 **Bad client credentials**)
   - 위치: **앱 > 플랫폼 키 > REST API 키** 페이지 **맨 아래 "클라이언트 시크릿"** → **카카오 로그인 활성화 OFF**
   - 끄면 `KAKAO_CLIENT_SECRET`은 **비워두면** 됨. (켜둘 거면 코드를 📋 **복사 버튼**으로만 입력 — `l`/`I` 오타 주의!)
7. (선택) **카카오 로그인 > 동의항목 > 닉네임** 설정
8. Vercel 환경변수: `KAKAO_REST_API_KEY`만 넣고, `KAKAO_REDIRECT_URI`·`KAKAO_CLIENT_SECRET`은 **비워두기** → **Redeploy**

**카카오 에러 → 해결 (실제로 겪은 것)**
| 에러 | 해결 |
|------|------|
| **KOE004** "서비스 설정 오류" | 3번 — 카카오 로그인 일반 > 활성화 ON |
| **KOE006** redirect_uri 불일치 | 4번 — `/api/auth/kakao/callback` + `www.` 유무 정확히 |
| **Bad client credentials** | 6번 — 클라이언트 시크릿 OFF (또는 코드 정확히 입력) |

### 7-5. 토스 결제 (선택)
> 💡 비워두면 **토스 공식 테스트 키**로 작동합니다. (실제 돈 안 빠짐 — 학습용으로 충분!)
실영업하려면 https://www.tosspayments.com 에서 본인 키를 발급(사업자 필요)해 `TOSS_CLIENT_KEY`/`TOSS_SECRET_KEY`에 입력.

### 7-6. 관리자 비밀번호
`.env`의 `ADMIN_PASSWORD`를 **본인만의 값**으로 꼭 바꾸세요. 이걸로 `/admin`에 로그인합니다.

### 7-7. 적용
`.env` 저장 → 서버 껐다 켜기(`Ctrl+C` → `vercel dev`) → `/setup`에서 전부 ✅ 인지 확인!

<a name="8"></a>
## 8. 인터넷에 올리기 (배포) + 도메인 연결

우리 사이트는 **Vercel(화면+서버리스) + Supabase(데이터)** 구조라 배포가 깔끔합니다.

### 8-1. GitHub에 코드 올리기
배포 서비스는 GitHub 창고에서 코드를 가져갑니다.
1. https://github.com 가입 → **New repository** (예: `saju-lab`) → Create
2. VS Code 터미널에서:
```powershell
git init
git add .
git commit -m "첫 업로드"
git branch -M main
git remote add origin https://github.com/내아이디/saju-lab.git
git push -u origin main
```
> ⚠️ `.env`는 `.gitignore` 덕분에 **자동으로 안 올라갑니다.** (정상!)

### 8-2. Vercel로 배포
1. https://vercel.com → GitHub로 가입 → **Add New > Project** → `saju-lab` 선택
2. **Environment Variables**에 `.env` 내용을 **하나씩 입력** (이름=값)
   - 단, **`BASE_URL`과 `KAKAO_REDIRECT_URI`는 배포 도메인 주소로 변경**
     (예: `https://saju-lab.vercel.app`, `https://saju-lab.vercel.app/auth/kakao/callback`)
3. **Deploy** → 잠시 후 완료! `https://...vercel.app` 주소로 전 세계 접속 🎉

> 🔑 배포 후 할 일: **카카오 콘솔에 새 Redirect URI 추가 등록**
> (`https://saju-lab.vercel.app/auth/kakao/callback`)
> 그리고 배포 사이트의 **`/setup`** 으로 들어가 전부 ✅ 인지 확인.

### 8-3. 도메인 연결 (간판 주소)
1. **도메인 구매**: 가비아 / 후이즈 / Cloudflare 등 (`.com` 연 1~2만원)
2. Vercel 프로젝트 → **Settings > Domains** → 산 도메인 입력
3. Vercel이 알려주는 값(보통 `A`/`CNAME` 레코드)을 도메인 산 사이트의 **DNS 설정**에 입력
4. 몇 분~몇 시간 뒤 내 도메인으로 접속됨!

> 🔑 **DNS** = 도메인(간판)을 실제 서버 주소로 **연결해주는 전화번호부**.

<a name="9"></a>
## 9. 자주 나는 오류와 해결법

| 증상 | 원인 | 해결 |
|------|------|------|
| `node`/`vercel` "명령어 못 찾음" | 설치 안 됨/터미널 안 껐다 켬 | 설치 후 **터미널 재시작** (`npm i -g vercel` 했는지) |
| `vercel dev` 빨간 에러 | 폴더 위치 틀림 / `npm install` 안 함 | 터미널이 `saju_make` 안인지, `npm install` 했는지 |
| `/setup`에서 **중앙 API ❌** | 대표 주소/키 미입력 or 중앙 미배포 | `SAJU_API_BASE`·`SAJU_API_KEY` 확인, 대표 서버 가동 확인 |
| `/setup`에서 **Supabase ❌** | URL/키 틀림 or 스키마 미실행 | 키 확인 + `supabase/schema.sql` RUN 했는지 |
| 분석 시 **"포인트가 부족"** | 가맹점 포인트 소진 | 대표에게 포인트 충전 요청 |
| 분석 시 **"OpenAI 키 미설정"** | `OPENAI_API_KEY` 비어있음 | 키 입력 후 서버 재시작 |
| 결제창은 뜨는데 승인 실패 | Supabase 미연결(주문 저장 필요) | `/setup`에서 Supabase ✅ 확인 |
| 카카오 로그인 에러 | Redirect URI 미등록 | 카카오 콘솔에 정확한 주소 등록 + 서버 재시작 |
| 키 바꿨는데 반영 안 됨 | 서버 재시작 안 함 | `Ctrl+C` 후 `vercel dev` 다시 |
| GitHub에 `.env`가 올라감 😱 | `.gitignore` 누락 | 즉시 키 재발급 + `.gitignore`에 `.env` 추가 |

> 💡 **오류가 나면**: 에러 메시지를 그대로 복사해서 **클로드 코드에게 "이 에러 고쳐줘"** 라고 붙여넣으세요.

## 10. 강사용 — 강의 진행 순서 (커리큘럼)

### 1일차 — 개념 & 환경 세팅
1. 0장: 결과물 시연 + **본사/가맹점(중앙 API + 포인트) 구조** 설명
2. 1~2장: 코딩 기초 + 용어(서버리스·Supabase·API키·포인트 위주)
3. 4장: 프로그램 설치 (Node, VS Code, Git, Claude Code) + `node -v` 체크

### 2일차 — 로컬 실행 & 진짜 분석
4. 5장: 파일 구조 (특히 `api/` 와 `supabase/schema.sql`)
5. 6장: `npm install` → `vercel dev` → **`localhost:3000`** 띄우기
6. 7장: **Supabase 만들고 schema.sql 실행** + `.env` 6칸 채우기 + 대표 API키 배포
7. ✅ 체크포인트: **`/setup` 전부 ✅** → 사주 입력 → **진짜 만세력 + AI 해설** 뜨면 성공! (가장 큰 순간)

### 3일차 — 운영 기능
8. 관리자(`/admin`): 상품·가격·프롬프트 수정, **배너/썸네일 이미지 교체**, 매출·방문 통계
9. (선택) 카카오 로그인 / 토스 실결제 키 연결

### 4일차 — 배포 & 도메인
10. 8장: GitHub 푸시 → **Vercel 배포** → 인터넷 주소 접속 🎉
11. 배포본 `/setup` 점검 + 카카오 Redirect URI 추가 + 도메인 연결
12. ✅ 체크포인트: 수강생 휴대폰으로 각자 사이트 접속·분석
